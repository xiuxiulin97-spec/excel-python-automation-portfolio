# Excel + Python 自動化工具開發者作品集路線

這份路線的目標不是長時間學理論，而是用 3 個月做出可以展示、可以接案、可以延伸的自動化工具作品集。

## 方向定位

你要培養的定位是：

> Excel + Python + 自動化工具開發者

目前先以 Excel、CSV、Python 自動化為核心。待 AI 客戶回饋摘要工具完成後，再升級為 AI + Excel + Python 自動化工具開發者。

優先接近的案件類型：

- Excel 自動整理
- Excel / CSV 報表生成
- 客戶資料清洗
- Google Sheets 自動化
- AI 摘要與分類
- 小型企業內部流程自動化

## 作品集安排

| 順序 | 專案 | 時間 | 難度 | 接案展示重點 |
| --- | --- | --- | --- | --- |
| 1 | Excel 自動報表工具 | 7-10 天 | 入門 | 把原始 Excel 自動清理並產生統計報表 |
| 2 | 客戶資料清洗工具 | 7-10 天 | 入門+ | 整理客戶名單、去重、檢查電話與 Email |

## Project 1：Excel 自動報表工具

狀態：第一版已完成。

商業價值：

- 幫中小企業整理銷售、訂單或客戶交易資料。
- 把人工 Excel 清理流程自動化。
- 可展示「選檔 -> 清理 -> 統計 -> 輸出報表」的完整流程。

第一版功能：

- 選擇 Excel 檔案。
- 讀取第一個工作表。
- 清除全空白列。
- 刪除完全重複資料。
- 依照 `客戶` 統計 `金額` 總和。
- 產生 `output/result_YYYYMMDD_HHMMSS.xlsx`。
- 輸出三個工作表：`原始資料`、`清理後資料`、`統計報表`。

後續可升級：

- 選擇工作表。
- 多工作表合併分析。
- 打包成 EXE。
- 增加圖表與 PDF。

## Project 2：客戶資料清洗工具

建議下一個專案做這個，因為它和第一個專案很接近，容易複用 pandas、openpyxl、tkinter，也很適合中小企業接案。

狀態：v1.0 已完成，專案位於 `customer-cleaner/`。

商業價值：

- 很多客戶名單來自表單、Line、電商、活動報名，格式常常很亂。
- 這個工具可以幫客戶把名單整理成可匯入 CRM、Email 行銷工具或業務追蹤表的乾淨資料。

第一版功能：

- 使用者選擇 Excel 或 CSV。
- 清除全空白列。
- 刪除重複客戶。
- 統一欄位名稱，例如 `姓名`、`電話`、`Email`、`公司`。
- 檢查 Email 格式。
- 清理電話中的空白、括號、破折號。
- 輸出 `cleaned_customers_YYYYMMDD_HHMMSS.xlsx`。
- 輸出工作表：`原始資料`、`清理後資料`、`問題資料`。

建議專案結構：

```text
customer-cleaner/
├── main.py
├── ui.py
├── cleaner.py
├── validator.py
├── exporter.py
├── requirements.txt
├── README.md
└── tests/
```

7-10 天執行節奏：

- Day 1：定義欄位規格與範例資料。
- Day 2：建立讀取 Excel / CSV 功能。
- Day 3：建立去重與欄位標準化。
- Day 4：建立 Email / 電話檢查。
- Day 5：建立輸出 Excel。
- Day 6：建立 tkinter GUI。
- Day 7：補測試、README、展示截圖。
- Day 8-10：修正錯誤與準備作品集說明。

## Project 3：Google Sheets 自動同步報表

商業價值：

- 很多小公司用 Google Sheets 當內部資料庫。
- 工具可把表單資料自動整理、統計，再回寫到同一份或另一份 Google Sheets。

第一版功能：

- 讀取 Google Sheets 資料。
- 清除空白列與重複資料。
- 產生客戶或月份統計。
- 回寫統計結果到指定工作表。
- README 說明 Google API 憑證設定。

建議時間：10-14 天。

## Project 4：AI 客戶回饋摘要工具

商業價值：

- 店家、課程、顧問、電商常有大量文字回饋。
- 工具可以幫客戶快速知道常見問題、正面回饋、負面回饋與待處理事項。

第一版功能：

- 讀取 Excel / CSV 的文字回饋欄位。
- 用 AI 產生每筆回饋分類。
- 彙整主要問題與建議行動。
- 輸出 Excel 報表。
- 保留人工可檢查的原始資料。

建議時間：10-14 天。

## 學習策略

每個專案都用同一個節奏：

1. 先做能跑的版本。
2. 再補測試。
3. 再寫 README。
4. 再準備展示說明。
5. 最後才考慮打包與部署。

不要先陷入大型架構、框架比較或 AI 理論。你的作品集價值來自「真的能幫客戶省時間」。
